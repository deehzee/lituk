# Web UI — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task.
> Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a browser-based study interface that reuses the existing
`lituk.review` session core, adds a "drill misses" mode, and provides a
small dashboard and missed-questions history.

**Architecture:** Worker thread per session bridges the synchronous
`run_session` loop to HTTP. The web layer implements the `UI` protocol
(`show_prompt` / `show_feedback` / `show_summary`) using a pair of queues,
with no changes to `scheduler.py`, `bandit.py`, or `presenter.py`. `Flask`
serves a handful of JSON API endpoints; the frontend is vanilla HTML + CSS +
JS using vendored Pico.css.

**Tech Stack:** Python 3.12, Flask ≥ 3.0, SQLite (`sqlite3`), Pico.css
(vendored), vanilla JS, `pytest` + Flask `test_client`.

**Design spec:**
`docs/superpowers/specs/2026-05-09-web-ui-design.md`

## Task 9: Frontend — static HTML + CSS + JS

**Files:** `app/lituk/web/static/` (all static assets)

Pages:

- `index.html` — home: start-session form (chapter checkboxes), due-today
  pill (fetched on load), links to `/dashboard` and `/missed`.
- `session.html` — active session: polls `/api/sessions/{id}/state` every
  300 ms; renders prompt / feedback / summary views.
- `dashboard.html` — stats: fetches `/api/dashboard` on load; renders
  tiles (per-chapter accuracy, recent sessions, weak facts, coverage,
  streak).
- `missed.html` — history: fetches `/api/missed` + filter controls;
  "Drill these" button posts to `/api/sessions {mode:'drill'}`.

CSS: vendor `pico.min.css` + `app.css` for grade-button colours, card
layout, and CSS bar charts.

JS (`app.js`): dispatches to per-page initialiser via
`document.body.dataset.page`; session driver handles state polling,
renders each state kind, posts answers/grades.

No unit tests for static assets; covered by the end-to-end test in Task 10.
